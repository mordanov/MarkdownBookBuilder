# Feature Specification: PDF Document Formatting Control

**Feature Branch**: `005-pdf-formatting`
**Created**: 2026-07-14
**Status**: Draft
**Input**: User description: "Управление форматом получаемого документа: заголовки разных уровней с другим шрифтом (размер и тип) и изменением цвета фона и текста; текст расширенный на весь лист А4; интерактивное оглавление с гиперссылками в PDF"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - Heading Style Customization (Priority: P1)

Автор книги хочет управлять визуальным оформлением заголовков H1–H6 в итоговом PDF: выбирать шрифт, его размер, цвет текста и цвет фона (подложки) для каждого уровня заголовка независимо.

**Why this priority**: Это основная жалоба — заголовки выглядят идентично Markdown-тексту. Визуальная иерархия критична для читаемости книги.

**Independent Test**: Можно проверить, создав книгу с заголовками H1–H3, сконфигурировав разные цвета и шрифты, и убедившись, что каждый уровень отображается с правильными стилями в PDF.

**Acceptance Scenarios**:

1. **Given** в конфигурации книги заданы стили для H1 (шрифт "Bold", размер 24pt, цвет текста белый, фон тёмно-синий), **When** выполняется сборка PDF, **Then** все заголовки H1 в PDF отображаются с этими стилями.
2. **Given** стиль для H2 задан (размер 18pt, без фона, цвет текста тёмно-серый), **When** выполняется сборка PDF, **Then** заголовки H2 отображаются без подложки с тёмно-серым текстом.
3. **Given** конфигурация не содержит переопределений стилей, **When** выполняется сборка PDF, **Then** применяются стили по умолчанию (разумная читаемая типографика).
4. **Given** задан некорректный цветовой код, **When** выполняется сборка, **Then** система сообщает об ошибке конфигурации с указанием проблемного поля.

---

### User Story 2 - Full A4 Page Width Text Layout (Priority: P2)

Автор хочет, чтобы текст в PDF занимал всю ширину листа A4 (с разумными полями), а не отображался в узкой колонке, как это происходит сейчас.

**Why this priority**: Сейчас текст выглядит как Markdown-документ с узкими колонками, а не как профессиональная книга. Это напрямую влияет на читаемость и профессиональный вид.

**Independent Test**: Можно проверить, измерив ширину текстового блока на странице PDF — она должна соответствовать ширине A4 за вычетом заданных полей.

**Acceptance Scenarios**:

1. **Given** конфигурация задаёт формат страницы A4, **When** выполняется сборка PDF, **Then** текстовый блок на каждой странице занимает полную ширину A4 минус поля (по умолчанию 2.5 см с каждой стороны).
2. **Given** конфигурация задаёт пользовательские значения полей, **When** выполняется сборка PDF, **Then** текстовый блок использует указанные поля.
3. **Given** документ содержит код-блоки и таблицы, **When** выполняется сборка PDF, **Then** эти элементы также растягиваются на полную ширину текстового блока.

---

### User Story 3 - Interactive PDF Table of Contents (Priority: P3)

Автор хочет, чтобы оглавление в PDF содержало кликабельные гиперссылки, ведущие на соответствующие страницы документа.

**Why this priority**: Оглавление без гиперссылок не позволяет навигировать по книге в PDF-ридере, что значительно снижает удобство использования.

**Independent Test**: Можно проверить, открыв PDF в Adobe Acrobat или аналогичном ридере и кликнув по записи в оглавлении — читатель должен перейти на соответствующую страницу.

**Acceptance Scenarios**:

1. **Given** книга содержит заголовки H1–H3, **When** выполняется сборка PDF, **Then** в оглавлении каждая запись является кликабельной гиперссылкой, ведущей на правильную страницу.
2. **Given** оглавление включает вложенные разделы, **When** пользователь кликает на запись подраздела, **Then** PDF открывается на нужной странице.
3. **Given** документ обновляется (изменяются страницы заголовков), **When** пересобирается PDF, **Then** все гиперссылки оглавления указывают на актуальные страницы.

---

### Edge Cases

- Что происходит, если заголовок содержит специальные символы или эмодзи?
- Как система ведёт себя при очень длинном заголовке, не умещающемся на одной строке?
- Что происходит при нулевых или отрицательных значениях полей?
- Как обрабатываются вложенные заголовки глубже H6?
- Что если цвет текста и цвет фона совпадают (текст становится невидимым)?

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: Система ДОЛЖНА позволять задавать стиль для каждого уровня заголовка (H1–H6) независимо в конфигурационном файле книги.
- **FR-002**: Для каждого уровня заголовка ДОЛЖНЫ быть настраиваемы: тип шрифта, размер шрифта, цвет текста, цвет фона (подложки за текстом).
- **FR-003**: При отсутствии пользовательской конфигурации стилей система ДОЛЖНА применять читаемые стили по умолчанию с визуальной иерархией.
- **FR-004**: Система ДОЛЖНА поддерживать формат страницы A4 и растягивать текстовый блок на полную ширину с учётом заданных полей.
- **FR-005**: Поля страницы (верхнее, нижнее, левое, правое) ДОЛЖНЫ быть настраиваемы в конфигурации книги.
- **FR-006**: Значения полей по умолчанию ДОЛЖНЫ соответствовать типографическим стандартам (2–2.5 см с каждой стороны).
- **FR-007**: Сгенерированный PDF ДОЛЖЕН содержать интерактивное оглавление с гиперссылками на страницы.
- **FR-008**: Гиперссылки оглавления ДОЛЖНЫ корректно работать в стандартных PDF-ридерах (Adobe Acrobat, Preview, Evince).
- **FR-009**: При пересборке книги гиперссылки оглавления ДОЛЖНЫ автоматически обновляться для отражения актуальных номеров страниц.
- **FR-010**: Система ДОЛЖНА сообщать об ошибке с указанием поля при некорректном значении в конфигурации стилей.
- **FR-011**: Система ДОЛЖНА поставляться с встроенной темой по умолчанию, обеспечивающей читаемую визуальную иерархию заголовков H1–H6 (без необходимости пользовательской настройки).

### Key Entities

- **HeadingStyle**: Стиль заголовка для конкретного уровня (H1–H6); атрибуты: уровень, семейство шрифта, размер, цвет текста, цвет фона, жирность, курсив.
- **PageLayout**: Параметры страницы; атрибуты: формат (A4/Letter), поля (верх/низ/лево/право), ориентация.
- **TableOfContents**: Конфигурация оглавления; атрибуты: включать/выключать, глубина заголовков, интерактивность.
- **DocumentTheme**: Совокупность настроек оформления книги, включающая HeadingStyle[] и PageLayout.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: Пользователь может настроить стиль всех 6 уровней заголовков менее чем за 5 минут, редактируя только конфигурационный файл.
- **SC-002**: 100% гиперссылок оглавления в сгенерированном PDF ведут на правильные страницы.
- **SC-003**: Текстовый блок занимает не менее 80% доступной ширины страницы A4.
- **SC-004**: Повторная сборка книги после изменения стилей занимает не дольше, чем первоначальная сборка.
- **SC-005**: Конфигурационные ошибки стилей обнаруживаются до начала генерации PDF и сообщаются с точностью до конкретного поля.

## Assumptions

- Экспорт PDF осуществляется через Typst (основной экспортёр согласно архитектуре проекта).
- Цвета задаются в формате HEX (#RRGGBB) или именованными значениями.
- Шрифты ограничены встроенными шрифтами Typst (без поддержки пользовательских или системных шрифтов в этой фиче).
- Конфигурация стилей размещается в TOML-файле конфигурации книги (уже используемый формат конфигурации) — отдельный файл темы не используется.
- Система поставляется с готовой темой по умолчанию: визуальная иерархия заголовков H1–H6 (убывающий размер, стилизованный фон для H1, без фона для H3–H6).
- Интерактивное оглавление генерируется автоматически из структуры заголовков — ручное редактирование оглавления не поддерживается.
- Формат страницы A4 является форматом по умолчанию; поддержка других форматов (Letter и т.д.) возможна, но не является требованием этой фичи.
